Overview

Postgres Database based tool for categorizing roads.

Outputs site categories to table op given junctions+crossings(table jc) and other events such as curvature and gradient.(table other_events)

How to Use:

Connect to a database

Import network layer and set fields:

The network is in table categorizing.network. Import this into qgis.

Under the layer tab: set network layer to this,label field to 'sec' and length field to 'meas_len'.

Selecting Section and Chainage

Sections can be selected by typing in the section label and pressing 'Go To' or by selecting a single feature of the network layer and clicking 'From Layer'. This will show and edit it's properties.

A marker will appear on the map corresponding to the chainage in the box. The 'From Click' button will cause clicking on the map to move the marker and chainage to the nearest point of the selected section

Input Junctions and Crossings

Junctions and crossings can be added at the selected section and chainage by selecting the category and clicking 'add'.

Junctions and crossings are stored in table categorizing.jc.Importing this into QGIS can be helpful

Altering the junctions and crossings table will automatically update the categories (in categorizing.op ).

Before and After Section:

Sometimes sections can contain the approach to a junction/crossing that is before or after the section.

Add junctions <50m after the end of a section with a chainage>section length.These will affect site categories for the forward direction.

Add junctions <50m before the start of a section with a negative chainage.These will affect site categories for the reverse direction.
